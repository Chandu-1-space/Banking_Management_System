## Banking Management System

# Project Overview

The Banking Management System is a MySQL-based database project designed to manage banking operations such as customer information, bank accounts, transactions, and loans. This project demonstrates database design, data management, and advanced SQL concepts commonly used in real-world banking applications.

# Features

- Customer Management
- Account Management
- Transaction Tracking
- Loan Management
- Primary and Foreign Key Relationships
- Joins
- Aggregate Functions
- Group By and Having Clauses
- Subqueries
- Views
- Stored Procedures
- Triggers
- Window Functions (RANK, DENSE_RANK)
- Business Analysis Queries

## Database Tables

# Customers

Stores customer information such as customer ID, name, phone number, and city.

# Accounts

Stores account details including account type and balance.

# Transactions

Stores deposit and withdrawal transaction records.

# Loans

Stores customer loan information.

## SQL Concepts Used

- DDL Commands (CREATE, ALTER, DROP)
- DML Commands (INSERT, UPDATE, DELETE)
- DQL Commands (SELECT)
- Joins
- Aggregate Functions
- Views
- Stored Procedures
- Triggers
- Window Functions
- Subqueries

# Sample Queries

- Find customers with account balances above average.
- Display top account holders based on balance.
- Calculate total bank balance.
- Generate account rankings using window functions.
- Retrieve customer account details using joins.
